## Blockers actuales

1. **`/etc/hosts` en cliente Windows** — el server tiene WireGuard (`10.66.66.2`)
   pero el cliente Windows no enruta hacia esa IP. Solución: en
   `C:\Windows\System32\drivers\etc\hosts` apuntar los hostnames a la IP LAN
   del server (`192.168.18.159`), no a `10.66.66.2`. Probado: `curl
   http://192.168.18.159:8081/` desde Windows responde (404 Traefik por falta
   de Host header, pero llega).

2. **Idioma default sigue siendo `es`** — la GS profile no acepta `ca` así que
   cookieplone instaló con `language_code="es"`. Cambio a `ca` requiere
   `plone.app.multilingual` install + setDefaultLanguage('ca') vía REST API
   o script. **Pendiente Day 1.5**.

3. **CI workflows del template** (`.github/workflows/{backend,frontend,
   docs,config,changelog}.yml`) están en el repo desde el scaffold pero
   no se han revisado ni ejecutado contra el código nuevo. Probable
   fallarán en lint hasta que añadamos `ruff` y `prettier` checks
   pasen — Day 1.5.

4. **`pnpm-lock.yaml` del frontend** sigue no commiteado. El
   Dockerfile.frontend usa `--no-frozen-lockfile` para que el build lo genere
   en cada vez. Day 2: hacer un primer `pnpm install` exitoso fuera y commitear
   el lockfile, luego restaurar `--frozen-lockfile`.

## Decisiones técnicas (desviaciones documentadas)

| # | Decisión | Razón |
|---|---|---|
| D-001 | `cookieplone` en vez de retired `cookiecutter-plone-starter` | El segundo no existe en GitHub |
| D-002 | Tooling Python en venv local + corepack para pnpm | Evitar sudo, mantener perfil del usuario limpio |
| D-003 | Idioma GS profile = `es` (no `ca`) | cookieplone no acepta `ca`. Catalan se añade post-install vía `plone.app.multilingual` |
| D-004 | Traefik en `:8081` (no `:80`) | Apache2 host ocupa :80 |
| D-005 | `setuptools<81` en `[container]` deps | setuptools 81+ extrae `pkg_resources` a paquete separado; Plone 6.1 lo usa por namespace imports |
| D-006 | RelStorage tables en schema `public`, no `plone_zodb` | Imagen Plone no lee `RELSTORAGE_SCHEMA_NAME` env; wiring `zope.conf` queda TODO Day 2 |
| D-007 | Dockerfile frontend: `pnpm install --no-frozen-lockfile` | No hay pnpm-lock.yaml committed aún; se generará en el primer build |
| D-008 | RelationChoice para `property_ref` con `selectableTypes=Property` | Restringe el target en el widget Volto |
| D-009 | Traefik **v3.7** (no v3.1 del template) | Docker 29 requiere API 1.44+, Traefik viejos solo hablan API 1.24 |
| D-010 | `DOCKER_API_VERSION=1.44` env en Traefik | Compatibilidad explícita con Docker 29.x |
| D-011 | Traefik `Host()` con `||` en lugar de comas | Traefik v3 requiere un argumento por Host(); coma rechazada |
| D-012 | Healthcheck Volto override con `--max-redirect=0` | Default healthcheck sigue 302→/ca→404 y marca unhealthy; aceptamos 2xx/3xx |
| D-013 | Frontend Dockerfile: skip `npm i -g corepack@latest` final | Falla por EACCES en `/usr/local/lib/node_modules`, hangs forever; builder stage ya tiene pnpm |
